# dotfiles リポジトリ - Claude Code向け指示

このリポジトリは、新しいMacで開発環境を自動構築するためのdotfilesリポジトリです。

---

## 🎯 リポジトリの目的

新しいMacを受け取った時に、`SETUP_GUIDE.md`に従ってセットアップすることで、開発環境を完全に復元します。

---

## 📁 ファイル構成と役割

### メインファイル
- **SETUP_GUIDE.md**: 詳細なセットアップ手順（必ず参照すること）
  - エラーハンドリング方法
  - よくあるエラーと対処法
  - 完了確認チェックリスト
- **README.md**: ユーザー向けドキュメント
- **PACKAGES.md**: インストールされるパッケージの詳細
- **MACOS_SETTINGS.md**: macOS設定の詳細

### スクリプト
- **setup.sh**: メインセットアップスクリプト
  - Homebrewパッケージのインストール
  - oh-my-zsh + プラグインのインストール
  - dotfilesのシンボリックリンク作成
  - macOS設定の適用（ユーザー確認後）
- **macos-defaults.sh**: macOSシステム設定スクリプト
  - Dock、Finder、キーボード、トラックパッドなどの設定
  - 実行前にユーザーに確認を求める

### 設定ファイル
- **Brewfile**: Homebrewパッケージリスト
- **.zshrc**: zsh設定（oh-my-zsh + プラグイン）
- **.gitconfig**: Git設定（個人情報は.gitconfig.localに分離）
### テンプレートファイル
- **.gitconfig.local.template**: Git個人設定のテンプレート
- **.env.template**: 環境変数のテンプレート

---

## ⚙️ セットアップの原則

### 1. 既存ファイルの保護
- 既存の設定ファイルは**必ずバックアップ**してから上書き
- バックアップ先: `~/.dotfiles_backup_YYYYMMDD_HHMMSS/`

### 2. ユーザー確認
以下の操作は**必ずユーザーに確認**してから実行：
- macOSシステム設定の適用
- 既存設定の上書き

### 3. エラーハンドリング
- エラーが発生したら、**詳細なエラーメッセージ**を表示
- 対処法をユーザーに提示
- 続行可能なら続行、不可能なら停止
- SETUP_GUIDE.mdの「よくあるエラー」セクションを参照

### 4. 対話的なセットアップ
- Git設定（名前、メールアドレス）はユーザーに確認
- 環境変数（GitHubトークン）が必要かユーザーに確認
- 不明な点は推測せず、ユーザーに質問する

## 🔐 セキュリティ

### Git管理されているファイル
このリポジトリはプライベートですが、以下のルールを厳守してください：

**✅ 含めてOK（既にコミット済み）**:
- ユーザー名（`x24ken`）
- プロジェクトディレクトリパス
- システム設定（Dock、Finderなど）
- キーバインド、色設定

**❌ 絶対に含めてはいけない**:
- パスワード
- APIキー、アクセストークン
- 秘密鍵（SSH鍵、GPG鍵）
- メールアドレス
- クレジットカード情報
- その他の認証情報

### Git管理外のファイル（重要）
以下のファイルは`.gitignore`で除外されており、**絶対にコミットしない**：
- `~/.gitconfig.local`: 個人のGit設定（名前、メール）
- `~/.env`: GitHubトークン等の秘密情報

これらのファイルはローカルのみに存在し、テンプレートファイル（`.template`）から作成します。

## 📝 セットアップ手順の概要

**前提**: ユーザーは既に以下を実行済み
- Command Line Tools のインストール
- Homebrew のインストール
- GitHub CLI のインストールと認証
- リポジトリのクローン完了（`~/dotfiles`ディレクトリに存在）
- `bootstrap-minimal.sh`の実行完了（Claude Codeがインストール済み）

1. **setup.shの実行**
   ```bash
   ./setup.sh
   ```
   - 確認プロンプトが複数回表示されます
   - macOS設定は任意

3. **個人設定ファイルの作成**
   - `.gitconfig.local.template` → `~/.gitconfig.local`
   - `.env.template` → `~/.env`
   - ユーザーに名前、メール、トークンを確認

4. **シェルの再起動**
   ```bash
   exec zsh
   ```

詳細は**SETUP_GUIDE.md**を参照してください。

## ⚠️ 重要な注意事項

### 1. NVMについて
- Node.jsのバージョン管理は**NVM**を使用
- プロジェクトごとのバージョン管理はNVMで行う

### 2. Python環境
- `pyenv`でPythonバージョンを管理
- `.zshrc`で`python`は`python3`にエイリアス

### 3. macOS設定
- システム設定を変更するため、必ずユーザーに確認
- 適用後、DockとFinderが自動再起動される

## 🛠️ エラー時の対応

### よくあるエラー
1. **Homebrewのエラー**
   - `brew update && brew doctor`を実行
   - エラーメッセージに従って修正

2. **oh-my-zshのインストール失敗**
   - 既存の`~/.oh-my-zsh`を削除
   - 再実行

3. **シンボリックリンクのエラー**
   - バックアップディレクトリを確認
   - 既存ファイルを削除
   - 再実行

詳細は**SETUP_GUIDE.md**の「トラブルシューティング」セクションを参照。

## 💡 カスタマイズ時の注意

### パッケージの追加
```bash
# パッケージをインストール
brew install <package-name>

# Brewfileを更新
brew bundle dump --force

# コミット
git add Brewfile
git commit -m "Add <package-name>"
git push
```

### macOS設定の変更
`macos-defaults.sh`を編集して、好みの設定に変更できます。
詳細は**MACOS_SETTINGS.md**を参照。

## 🎯 セットアップ時の目標

ユーザーがこのリポジトリを使って新しいMacをセットアップする際、以下を達成してください：

1. ✅ エラーなくセットアップが完了する
2. ✅ ユーザーが何が起きているか理解できる（透明性）
3. ✅ エラーが発生しても、適切に対処できる（ガイダンス）
4. ✅ 個人設定ファイルが正しく作成される
5. ✅ 全ての設定が反映される
